# Requirements: NoCode-AI v1.0

**Defined:** 2026-03-25
**Core Value:** Bots capture and nurture leads automatically; Felix closes the qualified ones personally.

## v1 Requirements

### Lead Notifications

- [x] **NOTF-01**: When a bot logs `action_type = 'lead'` to `bot_activity`, an email is automatically sent to felix@nocode-ai.co — **Validated in Phase 1** (actual recipient: feelo111295@gmail.com — felix@nocode-ai.co has no inbox)
- [x] **NOTF-02**: Email includes lead's name, business name, the message they sent, and which bot captured them — **Validated in Phase 1**
- [x] **NOTF-03**: Email is sent via Resend API from `bot_agent.mjs` on the DigitalOcean VPS — **Validated in Phase 1**
- [x] **NOTF-04**: Email sends reliably without blocking bot message processing (async, non-fatal on failure) — **Validated in Phase 1** (fire-and-forget .catch() pattern)

### Lead Tracking

- [x] **LEAD-01**: A `leads` table exists in Supabase with fields: id, created_at, user_id (Felix's), name, business, message, bot_name, status, notes — **Validated in Phase 2**
- [x] **LEAD-02**: When a bot captures a lead, a row is inserted into the `leads` table automatically — **Validated in Phase 2** (insertLead() deployed to all 5 bots)
- [x] **LEAD-03**: Felix can view all captured leads in a dedicated section of `dashboard.html` — **Validated in Phase 2**
- [x] **LEAD-04**: Felix can update a lead's status (New → Contacted → Closed → Lost) — **Validated in Phase 2** (inline dropdown, PATCH to Supabase)
- [x] **LEAD-05**: Lead list is sorted by newest first and shows name, business, bot, status, and time captured — **Validated in Phase 2**

### Client Onboarding

- [x] **ONBD-01**: When a client with `onboarded = false` logs in, they see a welcome screen instead of a blank/empty dashboard — **Validated in Phase 3**
- [x] **ONBD-02**: Welcome screen displays which bots are active and which platforms they're connected to — **Validated in Phase 3** (4 bot cards with Discord + Telegram badges)
- [x] **ONBD-03**: Client can dismiss the welcome screen, which sets `onboarded = true` in their Supabase profile — **Validated in Phase 3**
- [x] **ONBD-04**: After dismissal, client lands on the normal active dashboard with bot activity visible — **Validated in Phase 3**

## v2 Requirements (deferred)

### Lead Tracking (future)

- **LEAD-06**: Lead detail view — click a lead to see full conversation context from `bot_activity`
- **LEAD-07**: Notes field — Felix can add follow-up notes to a lead
- **LEAD-08**: Follow-up date — set a reminder date on a lead

### Notifications (future)

- **NOTF-05**: SMS notification to Felix when a high-intent lead is captured
- **NOTF-06**: Daily digest email summarizing new leads

### Onboarding (future)

- **ONBD-05**: Self-serve signup flow — clients sign up without Felix manually creating accounts
- **ONBD-06**: In-app bot configuration — clients set tone, platforms, and preferences themselves

## Out of Scope

| Feature | Reason |
|---------|--------|
| Self-serve client signup | Felix closes deals personally for now — not worth automating |
| In-app Stripe billing flow | Manual billing management sufficient for current scale |
| Bot configuration UI | Felix sets this up manually with each client |
| Mobile app | Web dashboard only for this milestone |
| Lead detail / conversation view | Deferred to v2 — status tracking sufficient for v1 |
