# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-25
**Phases:** 1–3
**Total Plans:** 4

## Overview

Three focused phases that close the gap between leads being captured and Felix being able to act on them — plus a clean first-login experience for new clients. Phase 1 wired up real-time email alerts so Felix stops missing leads. Phase 2 persisted those leads with status tracking so Felix can manage follow-ups. Phase 3 ensured new clients land on a welcoming dashboard instead of a blank screen.

## Milestone Summary

**Key Decisions:**

- Resend for email notifications — simple Node.js API, free tier, already had RESEND_API_KEY on VPS. Sends to feelo111295@gmail.com (felix@nocode-ai.co has no inbox).
- Leads stored in dedicated `leads` table — enables status tracking, notes, and follow-ups separate from `bot_activity`
- FELIX_USER_ID read from process.env — UUID never hardcoded in source
- insertLead follows logActivity pattern exactly — consistent fetch POST, no new abstractions
- Persistent leads section (not tab) — always visible below activity feed without clicking
- ACTIVE_BOTS array for onboarding display — no per-client DB query needed since all clients share the same 4 bots
- Overlay reuses loading-overlay pattern — consistent codebase, no new layout paradigm

**Issues Resolved:**

- bot_agent.mjs was outside the git repo (Desktop path) — copied into nocode-ai repo root, both kept in sync
- Supabase leads table pre-existed — idempotent DDL absorbed it gracefully
- SSH/SCP from executor environment — returned checkpoint:human-verify, Felix deployed from his machine with SSH key

**Issues Deferred:**

- businessName 'Not provided' stub — Discord carries no business metadata; structured capture deferred to v2
- Credentials hardcoded in HTML — acceptable per constraints; address security in future phase
- Self-serve client signup — Felix closes deals manually; automate in future milestone
- Lead detail / conversation view — status tracking sufficient for v1; link to bot_activity deferred to v2

**Technical Debt Incurred:**

- Tab nav CSS kept as dead code in dashboard.html (no tab HTML or switchTab JS)
- bot_agent.mjs duplicated: Desktop (VPS source of truth) + nocode-ai repo root (version control copy)
