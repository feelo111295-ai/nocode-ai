```javascript
async function notifyLeadCaptured({ botDisplay, userName, messageText }) {
  const subject = `New lead captured by ${botDisplay}`;
  const html = `
    <h2>New Lead Captured</h2>
    <p><strong>Bot:</strong> ${botDisplay}</p>
    <p><strong>Lead name / user:</strong> ${userName || 'Unknown'}</p>
    <p><strong>Message:</strong></p>
    <blockquote style="border-left:4px solid #ccc;padding-left:12px;color:#444;">
      ${messageText ? messageText.replace(/</g, '&lt;').replace(/>/g, '&gt;') : '(no message)'}
    </blockquote>
    <p style="color:#888;font-size:12px;">Sent by NoCode-AI bot runtime</p>
  `;
  try {
    await sendEmail({ to: 'felix@nocode-ai.co', subject, html });
    log(`Lead notification sent to felix@nocode-ai.co`);
  } catch (err) {
    log(`Lead notification failed (non-fatal): ${err.message}`);
  }
}
```

Key constraints:
- The try/catch ensures any Resend failure is non-fatal (per NOTF-04)
- `botDisplay` is passed in — use `config.display` at the call site
- `userName` is the Discord user who sent the message (from `meta.user` at the call site)
- `messageText` is the original user's message (from `content` parameter of handleMessage, passed at the call site in Task 2)
- HTML-escapes `<` and `>` in messageText to prevent broken HTML
